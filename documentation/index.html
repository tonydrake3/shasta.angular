Introduction

Material is a fully responsive admin web App built with Bootstrap & Angular. It uses Sass CSS which makes it easy to modify.

File Structure

Within the download you'll find the following file structure and contents

Once downloaded, unzip the compressed folder and you'll see something like this:

material/
├── material/                       ───> Contains the develop version of Material Admin Template.
│   ├── src/                           └──> All front end files are in this folder.
│   │     ├── app/                            └──> All the custom JS scripts and HTML markup
│   │     ├── assets/                         └──> All the assets, including images, fonts
│   │     ├── styles/                         └──> All the custom stylesheets (Scss files)
│   │     ├── vendors/                        └──> Third party libaries
│   │     ├── favicon.ico
│   │     ├── index.html
│   │     ├── main.ts
│   │     ├── polyfill.ts
│   │     └── vendor.ts
│   ├── config/                    ───> Contains the configuration files of Webpack
│   ├── package.json
│   ├── tsconfig.json
│   ├── tsconfig.webpack.json
│   ├── webpack.config.js
│   └── yarn.lock
├── CHANGELOG.md                   ───> The change log
├── dist/                          ───> The distribution (production) version of Material Admin
├── documentation/                 ───> Where the help documentation is.
└── Preview.png                    ───> The preview image

Introduction to some important files:

tsconfig.json
is the configuration JSON file for TypeScript
package.json
is the JSON file of NPM (You can click the link to know more). This file is used by npm to store metadata for projects published as npm modules.
webpack.config.js
is for Webpack, look in ./config folder for more
yarn.lock
is for Yarn users, so that others can get precisely the same dependencies. (If you don't wanna use Yarn, simply ignore it and keep using NPM)

How to Run the Demo?

Note: dist/ folder contains the built distribution (production) version, which is for demo purpose only

You need to put the files on a server to run it. Here we told about how to build a server locally.

You can build a locally server using WAMP or MAMP (if you uses PHP). Also it's pretty easy to build one with Python or Ruby (if you prefer Python or Ruby). And here we'll build a local server with NodeJS

Preparation:

You'll need to install Node.js >=v6.9.0 (NPM comes along with it) and node-static (a NPM package)


Steps:

It's easy if you are familiar with Node. Download Node.js here then install it, and install node-static package with one command line

$ npm install -g node-static

The usage of node-static is simple. Just enter the "dist" directory with command line

$ cd dist/

and type:

$ static

It'll return something like this: 

serving "." at http://127.0.0.1:8080

Then you can view the template on http://127.0.0.1:8080 with your browser

How to Use It?

In order to do development with it, first you'll need have the following tools installed:

Those tools are for development purpose only, you don't need them in production environment, meaning you can use any back-end languages you like, NodeJS is just one of the options, not required

  • Node.js >=v6.9.0 (NPM comes along with it), just download and install it, that's it.
  • Ruby (if you are using a Mac, you already have it), Sass for Scss CSS.

    • A few steps to get started:

    1. Enter "material" folder with your command line tool.

    $ cd material

    2. Install all the NPM packages with command: 

    $ npm install

    3. Then type npm start and you are ready to go :)

    $ npm start

    4. It will open the admin web app automatically with your default browser, if not, go to http://localhost:3003 in your browser (Note: The first time, it may takes a bit longer to load, about 20 seconds)


    Note: All the source code for development are inside ./material/src/ folder.

    Angular CLI

    This guide is only for people who want to use Material admin with Angular CLI. If you don't want to use Angular CLI, simply ingore it and use the default Webpack solution.

    Check out CLI Quickstart first to know the basis.

    Note: Special thanks to two buyers of Material admin - @DidarB, @LeFerdi - for putting up this guide!

    To use Material admin with Angular CLI (tested with v1.0.0):

    1. Make sure you have installed Angular CLI globally with command: npm install -g @angular/cli (To install a specific version you can use command like
      npm install -g @angular/cli@1.0.0
      )
    2. Create a project using Angular CLI with command: ng new YourProjectName --style=scss
    3. Replace default
      .angular-cli.json
      and
      package.json
      with files inside material/angular-cli folder
    4. install Material NPM packages by running command: npm install
    5. Replace folders and files under
      YourProjectName/src
      with material/src/app, material/src/assets, material/src/styles, material/src/vendors, material/angular-cli/index.html
    6. open
      ./src/app/app.component.ts
      and add import * as jQuery from 'jquery'; to the top of the file
    7. Run ng serve
    8. Done :)
